1 Create 4 GitHub Repositories

Sign into github.com as iaBRU-Web. Click the + icon top-right → New repository four times:

Repository nameVisibilityAdd a README?Why

nexus-adminPublicNo — leave untickedAdmin panel code
nexus-ai-db⚠️ Private⚠️ Yes — tick thisYour permanent database
nexus-ai-db must be set to Private. It stores hashed passwords. Anyone who reads it could attempt to crack them.

"Add a README file" must be ticked for nexus-ai-db. Without it the repo has no branch and the app cannot write files to it. The first signup will fail silently.

2 Create a GitHub Personal Access Token

This lets your backend and admin apps write user data to your private nexus-ai-db repo.

  1. Click your profile picture → Settings
  2. Scroll to the very bottom of the left sidebar → Developer settings
  3. Personal access tokens → Tokens (classic)
  4. Generate new token → Generate new token (classic)
  5. Note: nexus-db-token   |   Expiration: No expiration
  6. Scopes: tick the top checkbox labelled repo (ticks all sub-items automatically)
  7. Click green Generate token button
  8. A token appears starting with ghp_ — copy it immediately, it shows only once
Paste it into a text file on your desktop right now. You will need it in Step 5.

How to Upload Files to GitHub (read before Steps 3–5)

The process for every single file:
  1. Open the GitHub repository in your browser
  2. Click Add file → Create new file
  3. In the filename box, type the exact path from the table (e.g. pages/chat.js)
    When you type / GitHub creates the folder automatically
  4. Open the matching file from your unzipped folder in any text editor
    (Windows: Notepad — right-click the file → Open with → Notepad)
    (Mac: TextEdit — right-click → Open with → TextEdit)
  5. Ctrl+A (Win) or Cmd+A (Mac) to select all → Ctrl+C / Cmd+C to copy
  6. Click inside GitHub's editor → Ctrl+V / Cmd+V to paste
  7. Scroll down → click Commit new file
  8. Repeat for the next file
Hidden files: Files starting with a dot (.eslintrc.json) may be invisible.
Windows: File Explorer → View → tick "Hidden items"
Mac: Finder → press Cmd + Shift + . (dot key)

9 Test Everything End to End

As a user

  1. Visit https://bru-gpt5.vercel.app/signup
  2. Create an account — any username (3–32 chars) and password (min 6)
  3. After signup you land on the chat page — type a message and press Enter
  4. First response: 15–30 seconds — the AI model (up to 1.5 GB) is downloading. Normal. Every message after is faster.
  5. Use the model picker in the top-right to switch between 6 AI models

Confirm database is storing data

  1. Go to github.com/iaBRU-Web/nexus-ai-db
  2. A users.json file should appear — open it to see your account
  3. After chatting, a chats/ folder appears with your conversations

As admin

  1. Visit https://gpt5-admin.vercel.app → enter your ADMIN_PIN
  2. You should see stats and your user in the list
  3. Click a user to see their conversations, clear their chats, or delete the account

Common Problems

Login gives "Network error" or just spins forever
The NEXT_PUBLIC_API_URL on the frontend is wrong or was not saved.
Fix: Vercel → bru-gpt5 → Settings → Environment Variables → check the value is exactly https://backend-bru.vercel.app (no trailing slash).
Then: Deployments tab → click ⋯ on latest deployment → Redeploy.
"Database read failed" on login or signup
GITHUB_DB_TOKEN is wrong/expired, or GITHUB_DB_REPO is wrong format.
Fix: Vercel → backend-bru → Settings → Environment Variables.
Repo must be exactly iaBRU-Web/nexus-ai-db — no https://, no github.com.
Vercel won't let me name the project "bru-gpt5" or "backend-bru"
Vercel project names are globally unique. If taken, try bru-gpt5-ai or backend-bru-app.
If you use a different backend name, update NEXT_PUBLIC_API_URL on the frontend to match.
Backend build failed — Go compile error
Ensure go.mod is in the root of nexus-backend (not in a subfolder).
Its content must be exactly two lines:
module nexus-backend

go 1.21
Also: api/login/index.go must be in its own api/login/ folder, and api/signup/index.go in api/signup/. They must NOT be in the same folder.
First AI message never comes back
Normal — the model (600 MB – 1.5 GB) downloads on first use. Wait up to 60 s and retry. Once downloaded it stays cached for the session.
Admin says "Unauthorized"
The PIN must exactly match ADMIN_PIN in Vercel → gpt5-admin → Settings. Check capitalisation and no extra spaces.

How to See Error Logs

  1. Vercel → click the failing project
  2. Deployments tab → click the latest deployment
  3. Functions tab → click the function name (e.g. pages/api/chat/send)
  4. Real-time logs appear showing the actual error
